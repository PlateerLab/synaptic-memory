\begin{abstract}
Modern knowledge-graph retrieval-augmented generation (KG-RAG) systems --
Microsoft GraphRAG, HippoRAG, LightRAG, Cognee, Mem0 with graph memory --
rely on large language models to extract entities, relations, and
community summaries at index time. This makes incremental updates (a)
expensive, (b) non-deterministic, and (c) fragile under change data
capture (CDC) workflows common in enterprise deployments. We present
\textbf{Synaptic Memory}, an open-source KG-RAG system that builds its
graph with structural and statistical signals only -- no LLM at index
time -- and we prove a \textbf{Top-K Set Invariance theorem}: for any
two cumulative ingest schedules producing the same final corpus, the
top-$k$ retrieved set is identical. An empirical run on Allganize
RAG-ko (200 docs, 200 queries) shows 98.5\% set agreement and $|\Delta\,\mathrm{MRR}| < 0.01$ between one-batch and ten-batch streaming ingests.
Orthogonally, we identify a language-specific FTS improvement -- dropping
Kiwi-surviving Korean question-form morphemes at query time only -- that
lifts FTS-only retrieval MRR on every Korean public benchmark we tested
(Allganize RAG-ko: $0.621 \to 0.743$; AutoRAG: $0.592 \to 0.692$;
PublicHealthQA KO: $0.318 \to 0.466$) with zero regression on English
HotPotQA. Full source and ablation scripts are open at
\url{https://github.com/PlateerLab/synaptic-memory}.
\end{abstract}
