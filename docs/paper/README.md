## Why this paper

Synaptic Memory's actual contributions, in the order they'd land in a
RAG/retrieval venue:

1. **Top-k streaming invariance** — a formal property, empirically
   verified: ``mode="cdc"`` produces the same top-k as
   ``mode="full"`` for every query at every checkpoint. This gives
   incremental indexing a reproducibility guarantee that
   LLM-extraction pipelines (GraphRAG, Cognee, HippoRAG) provably
   cannot offer, because those pipelines' graphs depend on
   stochastic LLM output.
2. **Query-time Korean morphological stripping** — a small, principled
   query-preprocessing change that lifted FTS-only MRR by +0.08 to
   +0.15 on every Korean benchmark we measured, with zero regression
   on English. Measurable, ablatable, language-agnostic in framework
   though Korean-specific in parameters.
3. **An LLM-free-indexing baseline strong enough to matter** — not
   winning every benchmark, but close enough on public Korean RAG
   suites (Allganize, AutoRAG, PublicHealthQA) that the "you need
   LLM to build a knowledge graph" assumption behind the 2024-2026
   GraphRAG wave needs revisiting.

## Authoring workflow

```bash
# 1. Prose in Markdown (fast iteration)
$EDITOR docs/paper/draft.md

# 2. Regenerate figures if the ablation rerun produced new numbers
python docs/paper/figures/ablation_bar.py          # release-over-release Korean
python docs/paper/figures/streaming_invariance.py  # batch vs. streaming ingest
python docs/paper/figures/tier1_benchmark.py       # English multi-hop vs. HippoRAG2

# 3. arXiv-ready PDF
pandoc docs/paper/draft.md \
    --bibliography=docs/paper/references.bib \
    --citeproc \
    -o docs/paper/streaming_rag.pdf

# 4. (optional) Import draft.md into streaming_rag.tex for the final
#    arXiv upload.
```
